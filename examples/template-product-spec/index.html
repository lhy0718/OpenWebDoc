Product specification

Document runtime release spec

A structured product spec for defining scope, constraints, release gates, and owner responsibilities without losing the readable document surface.

Problem

Readers need to open a document, correct small details, and export safely.

Non-goals

No in-browser model calls, provider keys, plugin execution, or hidden instructions.

Success

Examples open from GitHub Pages, validate locally, and remain package-local.

Functional requirements

Capability Runtime behavior Validation rule
Open package Show document first, with minimal floating controls. Manifest and resources must pass validation.
Edit mode Allow small text and object corrections on the document surface. Requires declarative editing metadata.
Export Create a fresh `.htmlx` file from the current package state. Exported package must validate before sharing.
OwnerRuntime maintainer
Release riskMedium until accessibility pass
FallbackUse CLI validation and unpacked edits
