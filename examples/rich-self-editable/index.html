OpenWebDoc Introduction

OpenWebDoc is an open document package project for documents that can be read directly in the browser, edited lightly on their own surface, and safely revised by external coding agents without becoming page-bound print artifacts.

5. How The Document Becomes The Editing Surface

Direct human editing

  • Titles and paragraphs are edited as contenteditable text blocks.
  • Images and shapes are selected, moved, and deleted through small document overlays.
  • Tables remain real HTML tables so structure and accessibility survive editing.

External agent editing

  • htmlx unpack exposes the package itself as the editing boundary.
  • Agents revise HTML, CSS, metadata, and assets directly.
  • Revisions must pass directory validation, pack, and package validation before distribution.

The Goal Of OpenWebDoc

HTMLX is not a viewer wrapper around a document. It treats the document itself as the reading surface, editing surface, and agent-editable unit. The central artifact of OpenWebDoc is therefore not a screen trapped inside a separate app, but a browser-native document package that can be opened, edited, inspected, validated, and repacked.
